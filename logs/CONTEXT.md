# logs/ - Application Logging & Audit Trails

## PURPOSE
Centralized logging directory for all PolyWeather application components with structured log analysis.

## LOG STRUCTURE
```
logs/
├── application.log          # Main application events
├── trading.log             # Trade execution and P&L tracking  
├── api.log                 # External API calls and responses
├── security.log            # Authentication and security events
├── database.log            # Database queries and performance
├── websocket.log           # Real-time data streaming events
└── error.log               # Error tracking and debugging
```

## LOG LEVELS
- **INFO**: Normal operations, trade executions, API calls
- **WARNING**: Rate limits, API delays, minor issues
- **ERROR**: Failed trades, API errors, database issues  
- **CRITICAL**: System failures, security breaches, data corruption

## ROTATION POLICY
- **Daily rotation**: Logs rotated at midnight UTC
- **Retention**: 30 days for regular logs, 90 days for security logs
- **Compression**: Older logs automatically compressed to save space
- **Backup**: Critical logs backed up to remote storage

## MONITORING INTEGRATION
```
Logs → Prometheus → Alerts → Dashboard → Team Notifications
```

## RELATIONS
- **Generated by**: All application services and Docker containers
- **Monitored by**: Prometheus metrics and alert system
- **Analyzed by**: Scripts in `/scripts/` for error tracking
- **Backed up with**: Database and configuration backups

## SECURITY CONSIDERATIONS
- Contains sensitive trading data - ensure proper file permissions (600)
- API keys and passwords are masked in logs
- Real-time monitoring for suspicious patterns

**Critical for debugging, compliance, and performance optimization**
